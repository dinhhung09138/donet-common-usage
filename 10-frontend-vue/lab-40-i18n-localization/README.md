# Lab: Internationalization (i18n) & Localization (Vue)

## Objectives

- Externalize all UI strings into locale resource files
- Implement runtime locale switching without a full page reload
- Handle pluralization, date/number/currency formatting per locale
- Support at least one RTL locale layout correctly

## Key Concepts

`vue-i18n` · `locale resource files` · `pluralization rules` · `Intl.DateTimeFormat/NumberFormat` · `RTL layout (dir attribute)`

## Tasks

- [ ] Set up vue-i18n with at least 2 locale resource files (e.g. `en`, `vi`)
- [ ] Replace all hardcoded UI strings on one full screen with translation keys
- [ ] Implement a locale switcher that updates the UI instantly without reload
- [ ] Handle a pluralized string (e.g. '1 item' vs '5 items') using the library's plural rules
- [ ] Format dates/numbers/currency using `Intl` APIs respecting the active locale
- [ ] Add a third locale using an RTL language (e.g. Arabic) and verify layout mirrors correctly

## Expected Output

A screen fully translated across 3 locales (including one RTL) with instant runtime switching, correct pluralization, and locale-correct date/number formatting.

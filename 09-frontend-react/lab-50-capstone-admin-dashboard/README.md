# Lab: Capstone: Admin Dashboard Mini-Project (React)

## Objectives

- Integrate routing, auth/RBAC, CRUD, data grid, charts, notifications, and theming into one cohesive app
- Apply consistent architecture (feature folders, typed API layer, shared UI kit) across the whole app
- Demonstrate the app end-to-end in a realistic admin scenario (users, orders, or inventory management)
- Produce a PR description and short ADR explaining the key architectural decisions made

## Key Concepts

`feature-folder architecture` · `integrated auth+RBAC+CRUD+grid+charts` · `design system consistency` · `PR description/ADR writing`

## Tasks

- [ ] Assemble a login-protected admin dashboard combining: nested layout+nav (labs 08-09), auth+RBAC (35-36), CRUD list+form+modal (18/20/24), notifications (25/37), charts (34), theming (39)
- [ ] Pick one realistic domain (e.g. order management or user administration) and model it end-to-end
- [ ] Apply the feature-folder architecture and typed API layer conventions established across earlier labs consistently
- [ ] Add at least one E2E test (from lab 47) covering the full admin journey
- [ ] Run a production build with the optimizations from lab 49 and deploy it
- [ ] Write a PR description and a short ADR documenting the state-management and architecture choices made for this capstone

## Expected Output

A single deployed admin dashboard app demonstrating every major pattern from labs 1-49 working together consistently, with a passing E2E journey test, a PR description, and an ADR.

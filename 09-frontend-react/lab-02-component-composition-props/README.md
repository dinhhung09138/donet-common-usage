# Lab: Component Composition & Props (React)

## Objectives

- Design components around single responsibility and explicit prop contracts
- Apply composition (slots/children) instead of prop-drilling flags for layout variants
- Type props strictly and document required vs optional inputs
- Build a small reusable UI kit (Button, Card, Badge) consumed by a feature screen

## Key Concepts

`props typing` · `children composition` · `render props` · `component API design` · `default props` · `controlled vs presentational components`

## Tasks

- [ ] Build a `Button`, `Card`, and `Badge` component with strictly typed props
- [ ] Use `children` composition to support arbitrary card content instead of a `content` prop
- [ ] Implement a scoped-content pattern (render prop or children-as-function) for a `List` component
- [ ] Split a screen into presentational components vs a container that owns state/data
- [ ] Add `displayName` and JSDoc for each exported component's prop contract
- [ ] Write a Storybook story or a manual demo page exercising every prop variant

## Expected Output

A small reusable component library where every component has a typed, documented prop/slot API and is exercised by at least one live demo screen.

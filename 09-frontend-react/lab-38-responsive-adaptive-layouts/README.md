# Lab: Responsive & Adaptive Layouts (React)

## Objectives

- Build layouts that adapt correctly across mobile/tablet/desktop breakpoints, not just shrink
- Implement a responsive data-table-to-card-list transformation for small screens
- Use container queries where they solve a problem media queries can't
- Verify touch-target sizing and tap-friendly spacing on mobile

## Key Concepts

`CSS Grid/Flexbox responsive patterns` · `container queries` · `breakpoint hook (useMediaQuery)` · `table-to-card transform` · `touch target sizing`

## Tasks

- [ ] Build a dashboard layout using CSS Grid that reflows from 3-column to 1-column across breakpoints
- [ ] Transform a data table into a stacked card list below a `md` breakpoint (not just horizontal scroll)
- [ ] Build a `useMediaQuery` hook and use it to conditionally render (not just CSS-hide) a mobile-only component
- [ ] Use a container query on a reusable widget so it adapts based on its container, not the viewport
- [ ] Audit and fix any touch target under 44x44px on the mobile nav
- [ ] Test the layout at 320px, 768px, 1024px, and 1440px widths and document any breakage found and fixed

## Expected Output

A dashboard that reflows correctly at 4 documented breakpoints, including a genuine table-to-card transformation and container-query-driven widget, not just CSS shrinking.

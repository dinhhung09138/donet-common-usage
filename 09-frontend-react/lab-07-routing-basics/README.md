# Lab: Client-Side Routing Basics (React)

## Objectives

- Set up client-side routing with static and dynamic (param-based) routes
- Read and write route params and query strings
- Implement programmatic navigation and a working browser back/forward experience
- Build a 404 / catch-all route

## Key Concepts

`react-router-dom` · `<Routes>/<Route>` · `useParams` · `useSearchParams` · `useNavigate` · `catch-all route`

## Tasks

- [ ] Install `react-router-dom` and configure a router with at least 4 routes
- [ ] Add a dynamic route `/items/:id` and read the param with `useParams`
- [ ] Add query-string state (e.g. `?sort=asc`) read/written via `useSearchParams`
- [ ] Implement programmatic navigation (`useNavigate`) after a form submit
- [ ] Verify browser back/forward correctly restores scroll position and route state
- [ ] Add a catch-all `*` route rendering a 404 page

## Expected Output

A multi-page app where direct URL entry, param routes, query strings, and back/forward navigation all behave correctly, including a 404 for unknown paths.
